================================================================================
GPIO PIN CONFIGURATION
================================================================================

IAMBIC PADDLE INPUT:
  GPIO 12 (Pin 32) ← DIT paddle
  GPIO 13 (Pin 33) ← DAH paddle
  GND (Pin 6)      ← Both grounds

3-POSITION SWITCH INPUT:
  GPIO 16 (Pin 36) ← TX Position (Transmit)
  GPIO 19 (Pin 35) ← OFF Position (system inactive)
  GPIO 20 (Pin 38) ← SPK Position (speaker routing)
  GND (Pin 6)      ← Switch grounds
  3.3V (Pin 1)     ← Switch power

PTT OUTPUT:
  GPIO 18 (Pin 12) → UV5R PTT Line (via relay recommended)
  GND (Pin 6)      → Common ground

AUDIO:
  ReSpeaker 3.5mm out → UV5R microphone input
  ReSpeaker speaker   → External speaker
  GND → All tied together

ALL PINS CONFIGURABLE IN WEB INTERFACE (no restart needed)

WHAT EACH COMPONENT DOES
================================================================================

RUST BACKEND (target/release/morse-backend):
- Monitors GPIO pins in real-time
- Detects iambic paddle input (DIT/DAH)
- Reads 3-position switch state
- Controls PTT output to UV5R
- Provides REST API on port 3030
- Logs all events to morse.log
- ~30-50MB compiled binary, uses 8-15MB RAM
- <1% CPU when idle
- Startup: <2 seconds

WEB INTERFACE (web/index.html):
- Pure HTML/CSS/JavaScript (no frameworks!)
- ~15KB uncompressed
- Configure GPIO pins dynamically
- Send morse code patterns
- View real-time system status
- Monitor event log (last 30 entries)
- Responsive design (works on phone/tablet)
- No build step needed
- Runs on port 8000

REST API (Backend on port 3030):
GET  /api/status    - Get current configuration and status
POST /api/config    - Update GPIO pin assignments
POST /api/morse     - Send morse code
GET  /api/log       - Retrieve event log
GET  /api/monitor   - System monitoring data

================================================================================
TESTING PROCEDURES
================================================================================

TEST PADDLE INPUTS:
  gpioget gpiochip0 12  # DIT paddle on GPIO 12
  gpioget gpiochip0 13  # DAH paddle on GPIO 13
  
TEST SWITCH POSITIONS:
  gpioget gpiochip0 16  # TX position on GPIO 16
  gpioget gpiochip0 20  # Speaker position on GPIO 20

TEST PTT OUTPUT:
  gpioset gpiochip0 18=1  # Activate PTT
  gpioset gpiochip0 18=0  # Release PTT

TEST AUDIO:
  arecord -l              # List recording devices
  aplay -l                # List playback devices
  speaker-test -t sine    # Test speaker output

TEST WEB INTERFACE:
  curl http://localhost:3030/api/status

================================================================================
SUPPORT & TROUBLESHOOTING
================================================================================

MOST COMMON ISSUES:

1. GPIO Permission Denied
   Fix: sudo usermod -a -G gpio pi && sudo reboot

2. ReSpeaker Not Found
   Fix: sudo apt install python3-pip
        git clone https://github.com/respeaker/seeed-voicecard.git
        cd seeed-voicecard && sudo bash install.sh && sudo reboot

3. Backend Won't Start
   Fix: cargo build --release
        Check port 3030: sudo lsof -i :3030

4. Web Won't Connect
   Fix: Check backend running: ./target/release/morse-backend
        Check firewall: sudo ufw allow 3030

5. Paddle Not Detecting
   Fix: gpioget gpiochip0 12 (manual test)
        Check wiring to GPIO 12
        Verify ground connections

6. PTT Not Activating Radio
   Fix: Test with LED first: gpioset gpiochip0 18=1
        Check relay is powered
        Verify UV5R PTT jack connections
